Flexbox - vad är det?


Flexbox är ett enkelt och som det låter flexibelt sätt att placera objekt i en kolumn eller rad. Man har en container som man ger egenskapen "flex". Då kommer boxarna att placeras från vänster till höger för att från början så gör flexbox så automatiskt. Man kan ädnra storlek på boxarna genom att ge de höjd och bredd. Man kan även välja att göra flexboxen till kolumn vilket gör att den blir en kolumn istället. Samma koncept som standard flexbox fast den linje som från början var horisontell blir nu vertikal. Ett bra sätt att arbeta om man vill göra en responsiv sida. Man kan även placera boxarna hur man vill på sidan, hur mycket plats de ska ta på varje rad eller kolumn så med flexbox kan man enkelt designa ett rutmönster och i varje ruta kan man lägga önskat innehåll t.ex. bilder.

DOM(Document Object Model) - Vad är det?

DOM är ett sätt att förklara att alla webbsidans delar är objekt. Vi jobbar oftast i olika språk(javascript, css, html) och dessa måste jobba tillsammans även ifall det är olika språk och där kommer DOM in. T.ex. med hjälp av javascript så kan man ändra utseendet på html-sidan, genom att med funktioner och/eller events(knappar, inlogg) få saker att dyka upp på html-sidan utan att behöva uppdateras. Så man kan säga att DOM och html är identiska men om något ändras i html så ändras det i DOM också, allt detta sker bakom kulisserna då webbläsaren sköter allt detta.

Fin kod - Vad menas med det begreppet?

Fin kod, snygg kod, strukturerad kod, kärt barn har många namn men något som är väldigt viktigt och oundvikligt när man skriver kod är att skriva den snyggt och strukturerat oavsett projekt så vet man aldrig hur lång tid det tar innan man ska jobba med det igen eller om det ens är en själv som kommer jobba med samma kod, om Malin ska ta över projektet där jag har skrivit koden så ska hon enkelt kunna hoppa in och fortsätta utan konstigheter. En bra regel man kan ha när man skriver kod är att lämna kommentarer om man är fler som jobbar på projektet för att enkelt beskriva vad det är som sker. Göra inhopp från kanten när man skriver i en funktion. Man kan även lämna mer mellanrum mellan vissa saker bara för att lättare se vad som hör ihop med vad.

Ramverk

Vad är ramverk egentligen? hur används de? och den viktigaste frågan av de alla, varför används de?
Ramverk i sig är mer eller mindre kod som någon annan redan har gjort som man kan återanvända sig av för att snabbare skriva kod och komma framåt i sitt projekt. Om man ska göra ett projekt så kollar man oftast på vilket ramverk som passar en, t.ex. för min del när jag gjorde min portfolio/cv uppgift tidigare i utbildningen så använde jag mig av Bootstrap som är ett ramverk. Där hade jag redan en navbar redo, ett gridsystem(likt flexbox), knappar m.m. allt finns redan klart och det som underlättar väldigt mycket med ramverk(åtminstone Bootstrap) är att allt är responsivt. Så sidan man gör med t.ex. Bootstrap blir responsiv från start. Med ramverket Bootstrap så får man tillgång till olika css och html koder för allt från knappar till formulär och det finns även javascript tillägg om man vill använda sig av det. Bootstrap är ett av de mest använda och mest kända ramverken.
Ett annat ramverk är React.
React skapades av facebookutvecklare och släpptes 2013 och nu används det av alla stora företag runtom i världen. Det som är speciellt med React är att det är ett Javascript ramverk. Man strukturerar koden i komponenter vilket gör den lätthanterlig och man kan återanvänta funktionalitet lättare. React är snabbt och populärt pga. snabba buggfixar, bra dokumentation, aktiv vidareutvekling och stor verktyglåda. Men som sagt så är React ett javascript ramverk och därmed kan andra funktioner hamna utanför så som datahantering eller annan funktionalitet så ibland så måste man ta in andra ramverk för att samarbeta med React. Men varför React är så bra och så populärt är ekosystemet kring det, det är öppen källkod och i ekosystemet finns det många andra komponenter som är lösningen för just det ändamålet du vill uppnå. React är högt efterfrågat på arbetsmarknaden och något som alla borde lära sig.
